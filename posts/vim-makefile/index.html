Vim Makefile无法输入tab问题

2023年4月14日
 | tool

问题描述

之前编写makefile都再IDE下，编辑器都有特殊设定，今天在编写一个简单编译命令时为了防止每次手工输入命令，在vim下写了个makefile，

运行，直接报错

提示第四行有问题？

第四行是一个命令，makefile要求命令以tab开头

而vim下输入tab

问问chatgpt看看大佬怎么回答

Q:vim 无法插入tab

A:如果你在 Vim 中无法插入 Tab 字符，可能是因为 Vim 将 Tab 键视为命令键而不是文本键。

为了解决这个问题，可以执行以下步骤：

打开 Vim，并进入插入模式。

输入 :set list 命令，以显示非打印字符。

按下 Tab 键，此时应该会显示一个类似于 ^I 的字符。

输入 :set noexpandtab 命令，以关闭将 Tab 键转换为空格的选项。

按下 Tab 键，此时应该会插入一个 Tab 字符。

输入 :set expandtab 命令，以重新启用将 Tab 键转换为空格的选项。

现在，你应该能够在 Vim 中插入 Tab 字符了。如果你仍然遇到问题，请确保你的终端设置正确，并且 Tab 键没有被绑定到其他命令上。

问题原因

默认情况下，vim不会将tab转换为空格，只有个人在home目录设置了.vimrc或.virc才会导致，无法输入空格的情况

解决方案

  1. insert模式下，输入ctrl+v, 再按tab完成一次tab输入，后续再输入需重复操作

  2. 一般模式下输入:set noexpandtab，再进入insert模式 按tab输入即可，编辑完成可通过:set expandtab恢复tab转空格功能

  3. 编辑home目录下.vimrc文件

    set expandtab
